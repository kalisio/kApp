# Introduction

The kApp includes all the necessary boilerplate that you will need to get started building your application:

  • client-side boilerplate (opens new window) in the root folder
  • server-side boilerplate (opens new window) in the api folder
  • continuous integration/deployment boilerplate (opens new window) in the root folder as Dockerfiles and Travis CI scripts

It also includes the minimum viable set of features to start:

  • a basic application layout (opens new window) including side navigation, application bar and a right panel
  • ready-to-go user authentication services (opens new window) and screens (opens new window)
  • a basic service (opens new window) to create/remove documents in database
  • a basic activity (opens new window) listing documents using a k-list (opens new window)
  • a basic editor (opens new window) to fill document properties when creating a new document
